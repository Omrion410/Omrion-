# ZK - Community Layout

---

# Purpose

This document defines the layout structure of the Community Screen.

The layout should encourage interaction while maintaining a clean and organized interface.

---

# Layout Philosophy

The Community should feel:

* Comfortable
* Friendly
* Dynamic
* Modern
* Organized

---

# Screen Structure

Top to Bottom:

* Safe Area
* Header
* Search Bar
* Stories
* AI Recommendations
* Categories
* Community Feed
* Trending Posts
* Suggested Communities
* Challenges
* Bottom Navigation

---

# Spacing

Horizontal Padding:

24 px

Vertical Section Spacing:

24 px

Card Gap:

16 px

---

# Feed Layout

Posts are displayed in a vertical scrolling feed.

Each post occupies the full available width.

---

# Stories Layout

Horizontal scrolling.

Circular avatars.

Maximum visible items:

10

---

# Categories

Horizontal chips.

Scrollable.

Selected category remains highlighted.

---

# Cards

Every card should have:

* Rounded corners
* Soft shadow
* Clean spacing
* Comfortable touch area

---

# Responsive Design

Phones:

Single-column layout.

Tablets:

Two-column feed where appropriate.

Desktop (Future):

Adaptive multi-column layout.

---

# Floating Action Button

Position:

Bottom Right

Primary action:

Create Post

---

# Empty Space

Maintain generous whitespace.

Avoid visual clutter.

---

# Accessibility

Support:

* Large Text
* Screen Readers
* High Contrast
* Reduced Motion

---

# Performance

Smooth scrolling.

Lazy loading for media.

Progressive image loading.

Maintain 60 FPS.

---

# Future Expansion

The layout should support:

* Live Rooms
* Events
* Community Marketplace
* Learning Spaces
* Group Discussions

---

# Design Principles

Elegant

Modern

Readable

Interactive

Scalable

Future Ready
