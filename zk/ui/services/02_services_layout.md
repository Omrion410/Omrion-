# ZK - Services Layout

---

# Purpose

This document defines the layout system of the Services Screen.

The layout must remain clean, flexible, and scalable while supporting future platform growth.

---

# Layout Philosophy

The Services Screen should prioritize:

* Clarity
* Simplicity
* Speed
* Personalization
* Consistency

---

# Screen Sections

Top to Bottom:

* Header
* Search Bar
* AI Recommendations
* Categories
* Featured Services
* Popular Services
* Recently Used
* Favorites
* All Services

---

# Spacing

Horizontal Padding:

24 px

Vertical Section Spacing:

32 px

Card Spacing:

16 px

Grid Gap:

16 px

---

# Grid Layout

Phones:

2 Columns

Large Phones:

3 Columns

Tablets:

4 Columns

Desktop:

5–6 Columns

---

# Service Cards

Each card contains:

* Icon
* Title
* Short Description
* Category
* Optional Badge

---

# Visual Hierarchy

Highest Priority:

* AI Recommendations
* Featured Services

Medium Priority:

* Categories
* Popular Services

Standard Priority:

* All Services

---

# Scrolling

Single vertical scrolling.

Horizontal scrolling is allowed only for category chips and featured service carousels.

---

# Empty Space

Maintain generous whitespace.

Avoid visual clutter.

---

# Loading State

Use skeleton placeholders.

Do not display empty areas while content loads.

---

# Responsive Behavior

All components resize automatically.

Cards maintain proportional dimensions.

---

# Future Expansion

The layout must support:

* New sections
* New service types
* Dynamic widgets
* Personalized dashboards

---

# Design Principles

The layout should always feel:

* Elegant
* Balanced
* Flexible
* Modern
* Easy to Navigate
