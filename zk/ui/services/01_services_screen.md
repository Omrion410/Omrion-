# ZK - Services Screen

---

# Purpose

This document defines the overall structure and behavior of the Services Screen.

The screen acts as the central gateway to all services available within the ZK platform.

---

# Screen Goals

The Services Screen should allow users to:

* Discover new services
* Access favorite services quickly
* Continue recent activities
* Search instantly
* Browse by category
* Receive personalized recommendations

---

# Layout Structure

Top to Bottom:

1. Header
2. Search Bar
3. AI Recommendation Section
4. Service Categories
5. Featured Services
6. Popular Services
7. Recently Used
8. Favorite Services
9. All Services Grid

---

# Header

Display:

* Services title
* User greeting (optional)
* Notification button

---

# Search Bar

Persistent search field.

Supports:

* Instant search
* Voice search (Future)
* AI search

---

# AI Recommendations

Display personalized suggestions based on:

* User interests
* Recent activity
* Goals
* Favorites

---

# Categories Section

Horizontal scrollable category chips.

Example categories:

* Sports
* Health
* Education
* Community
* AI
* Productivity
* Entertainment

---

# Featured Services

Highlight selected services.

Large premium cards.

---

# Popular Services

Display trending services.

Automatically updated.

---

# Recently Used

Quick access to the user's latest services.

---

# Favorite Services

Pinned services selected by the user.

---

# All Services

Responsive grid containing every available service.

---

# Empty State

If no services exist:

Display:

* Illustration
* Friendly message
* Refresh button

---

# Design Principles

The screen must always feel:

* Clean
* Modern
* Fast
* Premium
* Personalized
