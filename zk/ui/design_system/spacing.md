# ZK Design System - Spacing

## Philosophy

Spacing creates clarity, balance, and visual comfort.

Every screen in ZK should feel organized, clean, and easy to scan.

Consistent spacing improves readability, usability, and overall user experience.

---

# Objectives

* Improve readability
* Increase visual consistency
* Reduce clutter
* Create balanced layouts
* Support responsive design

---

# Spacing Scale

The design system follows an 8-point grid.

Base values:

* 4 px
* 8 px
* 12 px
* 16 px
* 24 px
* 32 px
* 40 px
* 48 px
* 64 px

---

# Page Margins

Mobile

16 px

Tablet

24 px

Large Screens

32 px

---

# Section Spacing

Between major sections:

32 px

Between related sections:

24 px

Between small groups:

16 px

---

# Card Spacing

Internal padding:

16 px

Space between cards:

16 px

Large featured cards:

24 px

---

# Button Spacing

Space between buttons:

12 px

Button internal padding:

16 px

Minimum touch area:

48 × 48 px

---

# Input Spacing

Label to input:

8 px

Input to helper text:

8 px

Between inputs:

16 px

---

# Typography Spacing

Title to subtitle:

8 px

Title to paragraph:

16 px

Paragraph spacing:

16 px

---

# Icon Spacing

Icon to text:

8 px

Icon groups:

12 px

Toolbar icons:

16 px

---

# Navigation Spacing

Bottom Navigation:

16 px

App Bar padding:

16 px

Drawer item spacing:

12 px

---

# Grid System

Use consistent alignment.

Prefer multiples of 8 px.

Avoid arbitrary spacing.

---

# Responsive Behavior

Spacing increases slightly on:

* Tablets
* Foldable Devices
* Desktop (Future)

---

# Accessibility

Maintain sufficient spacing for:

* Large text
* Screen readers
* Comfortable touch targets
* Reduced accidental taps

---

* Consistent
