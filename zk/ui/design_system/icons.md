# ZK Design System - Icons

## Philosophy

Icons in ZK should be simple, modern, recognizable, and consistent.

They should help users understand actions quickly without relying entirely on text.

Icons must never create confusion.

---

# Design Style

The icon system follows:

* Minimal Design
* Rounded Edges
* Consistent Stroke Width
* Modern Appearance
* Clean Geometry

# Colors

Icons should inherit colors from the Design System.

Primary actions use the primary color.

Errors use the error color.

Warnings use the warning color.

Disabled icons use neutral gray.

---

# Accessibility

Every icon must include:

* Accessibility Label
* Tooltip when appropriate
* Minimum touch target size

Icons must never be the only way to communicate important information.

---

# Animation

Icons may support:

* Rotation
* Fade
* Scale
* Pulse
* Ripple

Animations should remain subtle and meaningful.
