# ZK Design System - Buttons

## Philosophy

Buttons are the primary interaction element in ZK.

They should feel modern, premium, and easy to recognize.

Every button must clearly communicate its purpose.

---

## Primary Button

Purpose:

Main action on every screen.

Examples:

* Continue
* Start
* Save
* Join
* Confirm

Characteristics:

* Filled background
* Rounded corners
* Medium shadow
* High contrast text

---

## Secondary Button

Purpose:

Alternative action.

Examples:

* Cancel
* Later
* Skip

Characteristics:

* Outline style
* Transparent background
* Rounded corners

---

## Text Button

Purpose:

Low-priority actions.

Examples:

* Learn More
* View All
* Forgot Password

Characteristics:

* No background
* Color changes on press

---

## Icon Button

Purpose:

Quick actions.

Examples:

* Search
* Share
* Settings
* Favorite
* Back

Characteristics:

* Circular touch area
* Smooth ripple effect

---

## Floating Action Button

Purpose:

Main floating action.

Examples:

* Create
* Add
* Start Chat

Characteristics:

* Circular
* Floating shadow
* Fixed position

---

## Disabled Button

Characteristics:

* Reduced opacity
* No interaction
* Clear disabled appearance

---

## Loading Button

While processing:

* Show loading spinner
* Disable multiple clicks
* Keep button size unchanged

## Accessibility

Buttons must:

* Be easy to tap
* Have sufficient spacing
* Maintain high contrast
* Support screen readers

---

## Animation

On press:

* Slight scale down
* Ripple effect
* Smooth return

---

## General Rules

* Only one Primary Button per section.
* Use Secondary Buttons for less important actions.
* Avoid placing too many buttons together.
* Keep labels short and clear.
